# rhyfmobot

Telegram-бот для сбора, хранения и просмотра рифм к словам.

## Что делает

- предлагает случайное слово из `words.txt`;
- принимает рифмы (через запятую) и сохраняет их в `rhymes.db`;
- позволяет добавить своё слово;
- показывает рифмы по команде `/r <слово>`;
- показывает статистику базы по команде `/stats`;
- присылает подписчикам дневные рифмо-челленджи в случайное время;
- поддерживает редактирование рифм через кнопку в интерфейсе.

## Структура

- `bot.py` - основной код бота (handlers, логика диалогов, работа с SQLite).
- `words.txt` - словарь слов для выдачи пользователю.
- `rhymes.db` - база данных SQLite с таблицей рифм.
- `database.py` - отдельный async-инициализатор БД (сейчас в основном коде не используется).

## Быстрый запуск

1. Создать окружение и установить зависимости:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2. Указать токен Telegram-бота через переменную окружения:

```bash
export API_TOKEN="ваш_токен_из_BotFather"
```

3. Запустить:

```bash
python bot.py
```

## Запуск в Docker (фон + автостарт)

1. Подготовить `.env` с токеном:

```bash
cp .env.example .env
# отредактируйте .env и укажите API_TOKEN
```

2. Собрать и запустить контейнер в фоне:

```bash
docker compose up -d --build
```

3. Проверка статуса и логов:

```bash
docker compose ps
docker compose logs -f
```

4. Остановка/запуск:

```bash
docker compose stop
docker compose start
```

`docker-compose.yml` уже содержит `restart: unless-stopped`, поэтому бот автоматически поднимется после перезапуска Docker/сервера. Данные (`words.txt`, `rhymes.db`) хранятся в локальной папке `data`.

## Команды в Telegram

- `/start` - начать работу;
- `/help` - показать справку;
- `/r <слово>` - показать сохранённые рифмы к слову;
- `/stats` - показать число слов и связей между ними;
- `/challenge_on` - повторно включить ежедневные рифмо-челленджи;
- `/challenge_off` - отключить челленджи;
- `/challenge_status` - показать время следующего челленджа;
- `/challenge_now` - получить челлендж сразу для проверки.

## Дневные челленджи

При первом взаимодействии с ботом подписка создаётся автоматически. Бот
назначает следующий челлендж на случайный момент дневного окна. По умолчанию
уведомления приходят с 10:00 до 21:00 по часовому поясу `Europe/Simferopol`.
После `/challenge_off` автоподписка больше не включается сама; для повторного
включения используется `/challenge_on`.

Для челленджа бот выбирает одно из наименее заполненных слов случайной выборки:
так постепенно пополняются слабые места словаря. Подписки и следующее время
отправки сохраняются после перезапуска Docker.

Окно отправки настраивается в `docker-compose.yml`:

```yaml
CHALLENGE_TIMEZONE: Europe/Simferopol
CHALLENGE_START_HOUR: 10
CHALLENGE_END_HOUR: 21
```

## Резервная копия БД

Создать консистентную копию SQLite-базы:

```bash
python backup_db.py
```

По умолчанию копия сохраняется в папку `data/backups`.

## Как хранятся рифмы

- для каждой введённой группы рифм создаются двусторонние связи:
  - `слово -> рифма`;
  - `рифма -> слово`;
  - связи между самими рифмами внутри группы.

## Известные проблемы

- старый токен мог попасть в историю/логи, его нужно отозвать в BotFather;
- нет автотестов;
- в репозитории лежит файл БД и большой `words.txt`;
- есть дублирование логики и смешение sync/async-подходов.
